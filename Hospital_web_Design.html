Website Design

Clinic Web Application

Project Requirement

  • User can log into the clinic application using the valid credentials.
  • As an unregistered clinic application user, user should be able to sign up for the Product Name application either as a patient or a provider in the Sign up page.
  • As a registered provider, logging into the clinic application would redirect user to the Provider dashboard.
  • User can access all the information relevant to profile.
  • As a registered provider, the user can view his daily schedule and appointment details in the Provider schedule page.
  • As a provider, the user can view appointment details of a scheduled appointment by clicking the appointment slot in the provider schedule page.

UX Process

Targated User
  • Doctors, Patients, Mid Age & Old Age.
Goal
  • Doctor/Patient - Easy login to the particular dashboard.
  • Check the schedule/ appointment for the day with minimum clicks.
  • Direct contact with the doctor/ patient if any urgency required.
Research
  • As per my research doctor wants to see their today schedule to plan their day accordingly.
  • Also, they want to check the details of the appointed patient.
  • The patient wants to check the nearest doctor from their location.
  • And book appoint for the doctor as per the doctor's availability.
My Aproach
  • Understand the requirements of the existing users.
  • I decided targeted users and researched on similar sites.
  • Research on competitor analysis.
